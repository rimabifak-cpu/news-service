# Мультиканальная система - Инструкция по обновлению

## 🎯 Что изменилось

Теперь сервис поддерживает **несколько Telegram каналов**:
- Каждый канал имеет свой токен бота и ID канала
- У каждого канала свой AI промт по умолчанию
- Источники привязаны к конкретным каналам
- Посты публикуются в тот канал, к которому привязан источник

---

## 📁 Структура изменений

```
Этап 1: Модель Channel и миграция БД
Этап 2: API для каналов (CRUD)
Этап 3: news_processor и telegram_service
Этап 4: Админ панель для каналов
```

---

## 🔄 Обновление на сервере

### 1. Применить миграцию БД

```bash
ssh root@195.133.31.34
cd /opt/news_service

# Обновить код
git pull origin main

# Применить миграцию
docker-compose exec news_service python migrate_multichannel.py
```

Миграция создаст:
- Таблицу `channels`
- Поле `channel_id` в `sources`
- Поле `channel_id` в `posts`
- Канал по умолчанию из текущих настроек `.env`

### 2. Перезапустить сервисы

```bash
docker-compose restart news_service scheduler
```

### 3. Обновить файлы в контейнере

Так как Docker использует volume для templates, нужно скопировать файлы:

```bash
# Копирование обновлённых файлов
docker cp /opt/news_service/templates/index.html news_service:/app/templates/index.html
docker cp /opt/news_service/static/app.js news_service:/static/app.js

# Перезапуск
docker-compose restart news_service
```

### 4. Очистить кэш браузера

Нажмите **Ctrl+F5** на странице админ панели.

## 📖 Как использовать

### 1. Создать новый канал

1. Откройте админ панель: `http://your-server:8001`
2. Перейдите в раздел **"Каналы"**
3. Нажмите **"Добавить канал"**
4. Заполните форму:
   - **Название**: Например "Новости"
   - **Telegram Bot Token**: Получите у @BotFather
   - **ID канала**: @your_channel или -1001234567890
   - **AI промт**: Промт по умолчанию для этого канала
   - **Позиция логотипа**: bottom-right, top-left, etc.
   - **Прозрачность**: 0.0 - 1.0

### 2. Добавить источник для канала

1. Перейдите в **"Источники"**
2. Нажмите **"Добавить источник"**
3. Выберите **канал** из dropdown
4. Заполните остальные поля

### 3. Настроить автопубликацию

Для источника включите **"Автопубликация"**:
- Посты будут публиковаться автоматически
- Используется AI промт канала (или источника, если указан)

## 🔧 API эндпоинты

### Каналы

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/channels` | Список каналов |
| POST | `/api/channels` | Создать канал |
| GET | `/api/channels/{id}` | Получить канал |
| PUT | `/api/channels/{id}` | Обновить канал |
| DELETE | `/api/channels/{id}` | Удалить канал |
| GET | `/api/channels/{id}/sources` | Источники канала |

### Источники

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/sources?channel_id=1` | Источники канала |
| POST | `/api/sources` | Создать (channel_id обязателен) |
| PUT | `/api/sources/{id}` | Обновить |

## 📊 Миграция данных

После миграции:
- Все существующие источники привязаны к каналу по умолчанию
- Все посты привязаны к каналу через источник
- Старые настройки `.env` используются для канала по умолчанию

---

## ⚠️ Важные замечания

1. **Боты должны быть администраторами** в соответствующих каналах
2. **Нельзя удалить канал с источниками** — сначала удалите источники
3. **При удалении канала** удаляются все связанные источники (cascade)

## 🐛 Решение проблем

### Ошибка "Канал не найден"

Проверьте, что миграция применена:
```bash
docker-compose exec postgres psql -U news_user -d news_db -c "\d channels"
```

### Источники без канала

Обновите источники через админ панель, выбрав канал.

### Посты не публикуются

Проверьте логи:
```bash
docker-compose logs -f news_service | grep "Автопубликация"
```
